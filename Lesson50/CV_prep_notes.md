# Key notes on CV preparation

1. Keep It Clear and Concise
Length: Aim for 1 page. Use 2 pages only if your previous professional experience clearly supports your transition into tech.
Layout: Use a clean, professional layout with ample white space. Prioritize readability with clear headings and consistent formatting.


2. Strong Personal Statement
The CV should start with a short, tailored profile that clearly connects the applicant to the specific job listing.
This section should not be a generic self-description such as: “I am passionate about frontend development and eager to learn.”
Instead, it should answer: Why are you relevant for this specific role?

A strong profile should include:
- Target role
  Example: Junior Software Developer, React Developer Intern, etc.
- Relevant technologies from the job ad
Example: React, TypeScript, Next.js, REST APIs, accessibility, testing.
- Evidence from projects or training
Example: built responsive apps, used API integration, created reusable components, deployed projects.
- Transferable career-switcher value
Example: customer communication, teaching, problem-solving, business understanding, attention to detail, teamwork.
- Fit for the company or product
Example: user-focused products, fintech, e-commerce, education, healthtech, SaaS.

Example:
Career-switching Junior Frontend Developer with hands-on experience building React and TypeScript projects, including responsive interfaces, reusable components, and API-based applications. My previous background in customer-facing roles has strengthened my communication, problem-solving, and ability to understand user needs. I am especially interested in this role because it focuses on building user-friendly digital products in a collaborative team environment.

### Formula
I am a [target role] with experience in [2–4 technologies from the job ad].
I have built [type of projects/features relevant to the role].
My previous background in [old industry/role] gives me [transferable strengths].
I am interested in this role because [specific company/product/team fit].


3. Highlight Relevant Skills
The skills section should not be a random list of everything you have seen or tried. It should clearly show the skills you genuinely have and that are relevant to the job you are applying for.

Main rule - Your skills list should be honest but targeted.

- Do not write skills you cannot explain in an interview.
- Do not copy every keyword from the job ad.
- Do not use the exact same skills order for every application.
- Put the most relevant skills for that specific job first.

Think of the skills section as a shop window: you are not changing what is inside the shop, but you choose what to display first depending on the job.

Before applying, read the job ad carefully and ask:

- Is this role mainly frontend, full-stack, testing-focused, or AI-related?
- Which technologies are mentioned in the job ad?
- Which of those technologies have I actually used?
- Which of my real skills should appear first?

For example, if the job ad focuses on React and TypeScript, put React and TypeScript near the top.
If the job ad mentions testing, make Jest and Playwright more visible.
If the job ad mentions full-stack development, include Next.js server-side features, MongoDB, Prisma, Auth0, and Stripe higher in the list.

### Recommended skills format
#### Example (generic):

Frontend:
HTML, CSS, JavaScript, TypeScript, React, Next.js, Tailwind CSS, responsive design, accessibility basics
Full-Stack Development:
Next.js server-side features, API routes/server actions, MongoDB, Prisma ORM, CRUD operations, data modeling basics
Authentication & Payments:
Auth0, protected routes, user authentication flows, Stripe Checkout, Stripe Products and Prices API, test-mode payment integration
Testing:
Jest, React Testing Library, Playwright, unit testing, component testing, basic end-to-end testing
AI & Developer Productivity:
Vercel AI SDK, AI-powered feature integration, structured prompting, AI-assisted debugging, Anthropic AI workflow training
Tools:
Git, GitHub, pull requests, code reviews, Vercel deployment, environment variables, Figma basics

#### Example: frontend-focused job

Frontend:
React, Next.js, TypeScript, JavaScript, HTML, CSS, Tailwind CSS, responsive design, accessibility basics
Testing:
Jest, React Testing Library, Playwright
Tools:
Git, GitHub, Vercel, Figma basics
Full-Stack Exposure:
MongoDB, Prisma, Auth0, Stripe Checkout

#### Example: junior full-stack job
If the job is more full-stack, write the skills like this:
Full-Stack Development:
Next.js, TypeScript, MongoDB, Prisma ORM, API routes/server actions, CRUD operations
Frontend:
React, Tailwind CSS, responsive design, accessibility basics
Authentication & Payments:
Auth0, Stripe Checkout, Stripe Products and Prices API
Testing:
Jest, React Testing Library, Playwright
Tools:
Git, GitHub, Vercel

### Soft skills
LinkedIn’s career-change guidance emphasizes reframing previous experience and highlighting transferable skills instead of only listing old job duties. You previous work experience section is a better place to highlight these skills.

#### Example skill mapping:
Teacher - Explaining, planning, patience, feedback
Nurse - Responsibility, detail, stress tolerance, documentation
Sales - Customer understanding, communication, negotiation
Accountant - Accuracy, systems thinking, compliance
Restaurant/hospitality - Speed, teamwork, service mindset
Project coordinator - Planning, stakeholder communication, delivery

Weak - Team player.
Strong - Collaborated with sales, support, and operations teams to solve customer issues under time pressure.

Weak - Good communication skills.
Strong - Communicated with 30+ customers daily, handled unclear requirements, and translated user needs into practical solutions.


4. Showcase Projects
For junior developers and career switchers, the projects section is one of the most important parts of the CV.
If you do not yet have professional developer experience, your projects are your main proof that you can build real applications.
Include 2–4 strong projects. It is better to show 2 well-built, well-explained projects than 6 small or unfinished ones.

#### Each project should include:
Project name
Short description
Tech stack
Live demo link
GitHub repository link
3–5 bullet points explaining what you built

##### Example
Full-Stack E-commerce Application
Live Demo: [link]
GitHub: [link]
Tech stack: Next.js, TypeScript, React, MongoDB, Prisma, Auth0, Stripe, Jest, Playwright, Vercel AI SDK

Built a full-stack e-commerce application with product listing, cart functionality, user authentication, and checkout flow.
Created product and price data through the Stripe API and integrated Stripe Checkout in test mode.
Modeled product, cart, user, and order data using MongoDB and Prisma ORM.
Implemented protected routes and user authentication with Auth0.
Added automated tests with Jest and Playwright to validate important user flows.
Integrated an AI-powered feature using Vercel AI SDK to improve the user experience.

A project should not only work on your computer. It should be easy for another person to understand and review. Before adding a project to your CV, make sure:

- The live demo works.
- The GitHub repository is public, unless there is a strong reason not to make it public.
- The README explains what the project does.
- The README includes setup instructions.
- The README includes screenshots or a short demo description.
- The code is reasonably clean and organized.
- There are no exposed API keys, secrets, or private credentials.
- The project name is professional.
- The project is something you can explain in an interview.


Open-source contributions are a bonus, not a requirement.
For a junior applicant a clean personal project with a good README, deployed demo, and readable code is often more useful than a small open-source contribution you cannot clearly explain.

5. Create a Portfolio Website
For junior developers, GitHub is more important than a personal portfolio website.
A portfolio website can be useful, but it is optional. A clean GitHub profile with strong projects, good README files, and working demo links is essential.

Recruiters and hiring managers want to see not only what you built but also how you built it.


6. Experience Section
For career switchers previous work experience is not something to hide. It can show maturity, responsibility, communication skills, teamwork, and problem-solving. However, do not describe your old jobs in too much detail. Rewrite your experience so it supports your transition into next role.

### What to include
Include previous jobs that show useful transferable skills, such as:
Communication, Customer understanding, Problem-solving, Teamwork, Responsibility, Planning and organization, Attention to detail, Working under pressure, Documentation, Learning new systems, Explaining complex information, Working with clients, users, or stakeholders.

### How to write previous experience
Do not only list duties. Focus on transferable value.

Weak example: Sales Assistant
Sold products to customers.
Worked at the cash register.
Helped in the store.

Stronger example: Sales Assistant
Communicated with customers daily to understand their needs and recommend suitable solutions.
Handled multiple tasks under time pressure while maintaining accuracy and service quality.
Collaborated with team members to solve customer issues and improve the shopping experience.

This sounds more relevant for a junior developer because it shows communication, problem-solving, teamwork, and user understanding.

### Important
If you have internships, freelance work, volunteer projects, or part-time development work, place these first.

For older or less relevant jobs, one short line is enough.
Example: 
Additional experience: Customer service, hospitality, and administration roles, 2016–2022.

7. Education and Certifications
The education section should show your relevant training, courses, and certifications.
For career switchers, this section helps explain how you moved into software development. However, keep it short. Your projects, skills, and GitHub usually matter more than a long list of courses.
### What to include
- Frontend or full-stack development programs
- University or vocational education
- Relevant online courses
- Technical certifications
- AI-related courses or certificates
- English or language education, if relevant

### Recommended format
Program / Degree / Course Name
School / Organization
Date or expected completion date
Short description, if useful

Example:
Full-stack AI Development Program
Code2Career AB
2025–2026

### Important: 
Only include certifications that are relevant to the job. Do not include every small or unrelated course. It is better to list 2–4 relevant certifications than a long list of weak ones.
Education and certifications should support your story, not replace your proof.


8. Tailor Each CV
Do not send the exact same CV to every company. Before applying, read the job ad carefully and adjust your CV so it clearly matches the role. This does not mean lying or adding skills you do not have. It means making your most relevant skills, projects, and experience easier to find. Tailoring your CV means showing the employer: “I understand this role, and here is the most relevant evidence that I could be a good fit.”

### How to tailor your CV

Look at the job ad and identify:
The job title
The main technologies
The required soft skills
The type of product or industry
The repeated keywords
The responsibilities mentioned in the role

Then adjust your CV by:
Reordering your skills section
Highlighting the most relevant projects
Updating your profile section
Choosing the most relevant bullet points from your previous experience
Using similar keywords from the job ad, if they are true for you

### ATS and keywords
Many companies use Applicant Tracking Systems, also called ATS, to scan CVs.
To make your CV easier to read for both ATS and humans:
- Use standard section headings like Skills, Projects, Experience, and Education
- Use the same important keywords from the job ad when they honestly match your experience
- Avoid complex layouts, tables, icons, and skill rating charts
- Do not overuse keywords unnaturally

Do not write keywords just to pass the system. If you write a skill in your CV, you must be ready to explain it in an interview.

9. Include Relevant Links
GitHub: Showcase code repositories and contributions.
LinkedIn: Provide a professional LinkedIn profile link.
Portfolio: As mentioned, include a link to their portfolio website.

10. Use Action-Oriented Language
Verbs: Start bullet points with strong action verbs like "Developed," "Implemented," "Designed," "Optimized."
Achievements: Focus on accomplishments rather than just duties. Quantify results when possible (e.g., "Improved page load speed by 30%").

14. Proofread and Optimize for ATS
Your CV should be easy to read for both humans and Applicant Tracking Systems, also called ATS.
ATS is software that some companies use to scan, organize, or filter CVs. However, your CV should not be written only for software. It still needs to sound natural and convincing to a real recruiter or hiring manager.
Before sending your CV, check:
Spelling
Grammar
Dates
Contact details
Links to GitHub, LinkedIn, and live demos
Company names
Technology names, such as Next.js, TypeScript, Auth0, Stripe, MongoDB, and Prisma

Small mistakes can make the CV look careless, especially for developer roles where attention to detail matters.

File Format: Submit the CV in PDF, unless otherwise specified.

File Name: make sure that the file name is clear and contain your name and surname, e.g.:
- JohnDoe_CV.pdf
- JohnDoe_FrontendDeveloper_CV.pdf
- JohnDoe_Resume_2026.pdf

15. Seek Feedback
Seek feedback and use it to continuously improve and refine their resumes.

16. Include a Cover Letter
Although not part of the CV, recommend writing a tailored cover letter to accompany the application, explaining their passion and fit for the role.

17. Showcase Problem-Solving via AI
AI skills are valuable in 2026, but they must be presented carefully. Do not make it sound like AI writes the code for you. Employers want developers who can think, understand, review, and take responsibility for their work. Use AI as a professional development tool, not as a replacement for your own knowledge.
You can show AI proficiency in three places:
- Skills section
- Projects section
- Profile or Experience section, if AI is especially relevant to the role

### Examples

Skills section:
Vercel AI SDK, AI-powered feature integration, structured prompting, AI-assisted debugging, test planning, documentation support, responsible AI-assisted development

Projects section:
- Integrated an AI-powered product recommendation feature using Vercel AI SDK.
- Used Vercel AI SDK to generate personalized product descriptions or shopping suggestions.
- Designed prompts and handled AI responses inside a Next.js application.

Profile section:
Only mention AI in the profile if the job ad mentions AI, automation, productivity, or modern development workflows.
"Junior Frontend Developer with hands-on experience building full-stack React and Next.js applications using TypeScript, MongoDB, Prisma, Auth0, Stripe, automated testing, and AI-powered features with Vercel AI SDK. Trained in structured AI workflows, prompt design, and responsible AI-assisted development."

18. Using AI for CV preparation
Many applicants now use AI to generate generic CV text. This can make CVs sound robotic and impersonal.

Use AI to improve clarity, identify missing keywords, check grammar, and prepare for interviews, but make sure the final CV sounds like you and reflects your real experience. Recent career guidance also warns that AI-generated CVs often become generic, while AI is more useful as an editor or refinement tool.
